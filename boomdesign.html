Crafting a dynamic design system

What was the project?

To speed up the design team's workflow, I built a design system of UI components.

My role

Visual and accessibilty audit, design system organisation, component creation.

Description

Boom is one of Hero Gaming's iGaming brands. It has a strong online presence in Europe.

The problem

As an older product, Boom didn’t have components built in Figma, so the design team was rebuilding each component every time a new user interface was needed. This surfaced a need to speed up the design and development processes.

The outcome

Figma design system and components that drastically sped up internal processes.

Discover

The team was using screenshots and wireframes for prototyping, which was becoming time-intensive. To optimise the team’s work, I proposed a design system that would allow us to design more efficiently and allow for quick experimentation. I took a step back to evaluate if it was the perfect timing. I analyzed the Hero roadmap to ensure my proposal aligned seamlessly with our long-term goals. Connecting with the development team, we discussed how the design system could integrate smoothly with existing projects. I began reading documentation and exploring other design systems for best practices.

Define

We began breaking down a product - Boom Casino - and analyzing the atomic structure of the user interface. This allowed us to understand the layers of components that we would need to create. When the Boom design system had been completed, the structure would be applied to the other products under the Hero Gaming umbrella.

A good design system balances simplicity and granularity. Most importantly, it needs to be usable and scalable. This allowed for easy modernization and futureproofing if new brands were added to the Hero portfolio.

Develop

We kickstarted designing the components. We considered inclusivity as a core value, as a product isn’t successful if it doesn’t consider the experience for people with permanent, temporary, or situational disabilities. We audited the current website for visual and technical accessibility errors based off of WCAG 2.1 guidelines and corrected them in the newly built design system. This means that moving forward, Boom will remain accessible to as many people as possible.

Deliver

On completion I gave a presentation to the design team, and access to the design system was shared. See the Figma file here.

Successes

The design system unifies design work by speeding up and simplifying a designer’s workflow, because designers are no long starting from scratch.
