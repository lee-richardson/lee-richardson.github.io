Improved the browsing experience

Lee

Methods ↴

UI elements, wireframing, prototype testing, competitive analysis, stakeholder interviews.

Description ↴

Hero Gaming wanted to add a feature to its Japanese brand, to help users find games easier.

The challenge

We realized that users needed help navigating our large game selection. Two designers worked together to make this happen.

The goal

Add a new feature that allows users to filter games and the companies that created them (called "providers").

The process

Understand

Because analytics showed us our Japanese players mostly used phones, we had to make sure that the feature worked smoothly on mobile devices. We also took into account that Japanese users are used to seeing a lot of information on the screen at once.

Research

We already used Fullstory software to track user analytics, so we had a large amount of qualitative data to dig through. We started looking at the flow of how users find games, and how many use the already existing search function. Access to screen recordings, also through Fullstory, helped us spot any moments of confusion or frustration.

To think about filtering, we looked at casino gaming, travel booking and e-commerce sites, to see if any common patterns emerged.

Competitor research helped us discover common patterns in game browsing.

Analyze

A common problem we found is that Japanese users don’t always want to type in English, as it adds another layer of thought which removes them from the gaming experience. Because most providers have English names (“Evolution Gaming”, “Blueprint Gaming”) we decided to use tiles with logos, instead of a dropdown with text content.

Sketch

I quickly drew some rough ideas about where we could put the filtering function and how it could work within the existing flow. By exploring different options, we were able to choose the most effective placement.

Wireframes and prototype

We gathered logos from all the game providers and created wireframes. To keep things simple for users, we showed only a few filtering options at a time, with a choice to expand the filter. Users could also search for games using keywords. We made sure that the search bar stood out so that users could find it easily.

Thinking through the feature with wireframes.

Wireframes showing the different states.

Test design and iterate

We tested the feature multiple times using prototypes, then ran an A/B test and followed the results using Fullstory analytics.

Result

Data showed that after the feature was added to the flow, users were able to find their favourite games faster.

The user can choose to combine the filter feature with a searchterm.
