Improved the browsing experience

Casino.Me

2022

Lee

Methods ↴

UI elements, wireframing, prototype testing, competitive analysis, stakeholder interviews.

Description ↴

Based on business and user requirements, we introduced a feature into the user flow to help users find games easier. The task was completed by working with another designer over a two-week agile sprint.

The challenge ↴

Users needed help navigating our large game catalogue. We needed a way to make browsing easier without drastically changing the user flow.

The goal ↴

Find the best way to add filter functionality into the user flow, which allows users to easily find games and the companies that created them (called "providers").

Discover

We set up time for requirements gathering, where we carried out stakeholder interviews to determine the return on investment (ROI) and key performance indicator (KPI). From using the Fullstory analytics software, we knew that around 85% of our players in Japan used smartphones. While we had search functionality, In terms of UX benchmarking we had to add filtering to give a smoother experience than that offered by our competitors.

I took screenshots during competitor analysis to determine any best practices in terms of design patterns.

We ran a competitor analysis session where we signed into other products and browsed for games. While most only used search functionality, a couple of products allowed filtering of their game catalogue. We also investigated how filtering functions outside of iGaming, focusing on travel booking and e-commerce industries. This was to see if any common design patterns and UX mental models emerged.

Define

We used the Fullstory analytics tool to observe video recordings of the user flow of browsing and searching for games in our product. This helped us form an empathetic approach to our design ideas. Access to screen recordings helped us spot any edge cases or moments of user confusion or frustration.

Develop

A couple of ideation workshops took place, with time spent on brainstorming and sketching rapid prototypes of flow ideas.

I quickly sketched ideas to consider flow and interactions.

We sketched ideas that aligned with:

  • • Fitt’s law — the further away a target is and the smaller the size of the target, the longer it takes for a user to complete a task.
  • • Hick’s law — the more options are available, the longer it will take a user to make a decision.
  • Cognitive load - the amount of mental effort required to use the feature should be low, and shouldn’t overwhelm or confuse users.

Still in the paper sketch phase, ideas began to form around feature interactions.

A common problem we found is that Japanese users don’t always want to, or can’t, type in English characters. Because most providers use English characters (“Evolution Gaming”, “Blueprint Gaming”) we decided to use tiles with logos, instead of a dropdown with text content.

I created low-fidelity digital sketches of how the filtering function could work within the existing flow. We took into account the mental model around filtering, so users could recognize that the filter should function the way it appears.

Low-fidelity digital sketches helped me explore visual hierarchy, design patterns and mental models of interactions.

We created wireframes, and experimented with adding a filter function, keeping in mind the law of proximity. To reduce the cognitive load during the user flow, we decided to initially display only a few tiles in the default state, with a choice to expand the filter and show all tiles. Users could also search for games using a searchbar. We made sure that the searchbar stood out in the visual hierarchy so that users could find it easily.

Notes on how the filtering feature fits into the user flow.

We tested the low-fidelity prototypes during a user testing workshop with five employees of Hero Gaming, who weren’t familiar with the product. The response showed that the user flow caused no confusion and allowed users to find games quicker than the current task-completion time.

These low-fidelity prototypes were guerilla tested by five people, to give us insights into user expectations around the flow.

Happy with the outcome of the user testing workshop, we began creating new components needed for the filter feature. These were incorporated into a master design system to speed up our workflow.

Work began on the UI. Filter states are shown here.

A high-fidelity prototype with notes on resetting the filter to a default state.

We created high-fidelity prototypes and tested the feature again in another user testing workshop, again with five employees of Hero Gaming unfamiliar with the product. Comparing workshop test times against the current data analytics, we confirmed that there was less time spent finding a game, and less cognitive load overall.

Deliver

Design was approved by product management and went through handoff to the developers, and after development we confirmed the flow with the QA team.

Result

After a month we analyzed the data analytics and results indicated that users were able to find their preferred games more quickly, and with less cognitive load. This met our key performance indicator (KPI of the task.
